# BabyFoot Go

## Vue d'ensemble du projet

BabyFoot Go est une application iOS native développée pour gérer l'ensemble des activités de babyfoot au sein d'un établissement scolaire. L'application permet d'enregistrer les joueurs, créer des matchs, saisir les résultats, et consulter des statistiques détaillées avec un classement automatique des meilleurs joueurs.

## Le défi

L'établissement scolaire avait besoin d'une solution mobile pour :
- Gérer les joueurs et les associer à leurs formations
- Organiser des matchs de babyfoot en 1v1 et 2v2
- Enregistrer les résultats et conserver l'historique
- Générer des statistiques et un classement automatique
- Fonctionner sans connexion internet (offline-first)

## Solution développée

### Fonctionnalités principales

**Gestion des joueurs**
- Enregistrement des joueurs avec nom, prénom et photo
- Association des joueurs à leur formation scolaire
- Profil détaillé avec statistiques personnelles
- Système de recherche et de filtrage

**Création et gestion des matchs**
- Matchs en 1 contre 1 (1v1)
- Matchs en équipes de 2 (2v2)
- Saisie des résultats en temps réel
- Possibilité d'ajouter une photo du gagnant ou du match
- Historique complet de tous les matchs

**Statistiques et classement**
- Podium des trois meilleurs joueurs
- Nombre de victoires, défaites et matchs joués
- Taux de victoire et statistiques détaillées
- Classement automatique mis à jour en temps réel
- Filtres par formation

**Architecture offline-first**
- Stockage local avec SwiftData
- Persistance SQLite automatique
- Aucune connexion internet requise
- Synchronisation automatique des données

### Architecture technique

**SwiftUI & iOS 17+**
- Interface 100% SwiftUI pour une expérience native
- NavigationStack pour la navigation fluide
- Animations et transitions natives iOS
- Design moderne suivant les guidelines Apple

**SwiftData & Persistance**
- Modèles de données : Joueur, Formation, Match
- Relations entre les entités (joueurs ↔ formations, joueurs ↔ matchs)
- Stockage local SQLite via SwiftData
- Queries optimisées pour les performances

**Modélisation des données**
- **Joueur** : nom, prénom, photo, formation associée
- **Formation** : nom, liste des joueurs
- **Match** : type (1v1/2v2), joueurs participants, score, date, photo

## Résultats obtenus

- ✅ **Interface intuitive** : Adoption rapide par les utilisateurs

## Défis techniques relevés

### Architecture offline-first avec SwiftData

Le plus grand défi était de créer une application entièrement fonctionnelle hors ligne avec une persistance locale robuste. SwiftData a permis de simplifier la gestion des données tout en offrant les performances d'une base SQLite.

### Gestion des matchs 1v1 vs 2v2

Développement d'une logique différente pour les matchs individuels et en équipe, avec calcul automatique des statistiques personnelles même dans les matchs d'équipe.

### Calcul dynamique du classement

Mise en place d'un système de calcul automatique du classement basé sur le nombre de victoires, avec mise à jour en temps réel après chaque match.

### Relations complexes avec SwiftData

Modélisation des relations entre joueurs, formations et matchs avec SwiftData, en assurant la cohérence des données et les suppressions en cascade.

## Architecture de code

### Modèles SwiftData

```swift
@Model
class Joueur {
    var nom: String
    var prenom: String
    @Attribute(.externalStorage) var photo: Data?
    @Relationship var formation: Formation?
    @Relationship(deleteRule: .cascade) var matchs: [Match]
}

@Model
class Formation {
    var nom: String
    @Relationship(inverse: \Joueur.formation) var joueurs: [Joueur]
}

@Model
class Match {
    var date: Date
    var type: TypeMatch // 1v1 ou 2v2
    var score1: Int
    var score2: Int
    @Relationship var joueurs: [Joueur]
    @Attribute(.externalStorage) var photo: Data?
}
```

### Navigation moderne avec NavigationStack

Utilisation de NavigationStack pour une navigation fluide et moderne, avec gestion de l'état de navigation et deep linking.
